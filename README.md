# HungrySnake说明文档&&思路设计稿

# 一、游戏简介  
本游戏是一款2D平面小游戏。在游戏中，玩家操控着一只会不断饥饿而导致的身体缩短的贪吃蛇，需要通过不断进食以增加身体长度保证存活并获得更高的得分。  

# 二、操作说明  
w:向上调头；  
a:向左调头；  
s:向下调头；  
d:向右调头；  
space:按住进行加速  

# 三、游戏规则  
···游戏目标···  
获取更高得分  
···失败条件···  
贪吃蛇身体因饥饿消耗殆尽；  
贪吃蛇蛇头撞击到蛇身；  
贪吃蛇蛇头撞击到地图边界（边界模式游玩）  
···基本机制···  
地图中会随机刷新带点数1~9的“食物”，吃掉食物后会为贪吃蛇增加对应点数的一格身体，并在地图上刷新出一个新“食物”  
贪吃蛇末尾身体的点数会随游戏时间逐渐减小，点数减小至0时该体节消失，上一体节成为新末尾并开始减小点数  
吃食物时会增加得分，增加的得分还与此时的身体总长度相关，身体总长度越长增加的得分越多  
贪吃蛇速度除了与玩家是否按住space相关外，还与得分和身体总长度相关，得分达到一定阶段贪吃蛇基本速度增加，同时总长度越大速度越高  

# 四、游戏界面说明  
···主界面···  
Score（左上）：记录栏，记录了最佳得分、最佳长度、上次得分、上次长度  
Mode（左下）：模式选择，可以在此选择游玩边界模式与无边界模式  
Start（中间）：开始游戏，进入游戏界面
Exit（右上）：登出，退出游戏（该功能在网页版表现为网页界面冻结）  
···游戏界面···  
Class（左上）：阶段栏，显示当前所处阶段  
CurrentScore（左中）：得分栏，记录当前得分与长度  
Home（左下）：返回主界面  
Pause（左下）：暂停，点击暂停游戏，再次点击恢复游戏  
Game（右）：游戏主界面  

# 五、特色玩法  
独特的饥饿游戏机制  
与身体长度挂钩带来的动态得分奖励系统  
速度与得分、体长相联系带来的动态难度系统

# 七、已知问题&&预期修复  
···初始点数可能随机到较小值，导致出现开局必死的局面···  
未来准备设置前n个“食物”为最大点数9，避免初始必死的现象出现  
···蛇头初始位置固定···  
未来准备设定蛇头位置随机  
···点击pause后使用home回到主界面，再次开始游戏时蛇处于暂停状态···  
这是因为蛇还处于pause状态未恢复，重新游戏刷新了pause图标，但未刷新蛇状态。此时点击两下pause可以恢复  
未来准备点击Start后刷新蛇状态来进行修复

# -----------------以下为思路设计稿------------------------  
Q：游戏的初衷是什么  
A：练手Unity  
Q：为什么要设计饥饿机制  
A：游玩传统贪吃蛇时有时感到枯燥，认为是因为游戏流程固定带来的重复引发的，因此，添加饥饿机制为游戏增加更多变量，实现每一局游戏都不可能一样。而且饥饿机制也符合贪吃蛇游戏的背景。  
Q：为什么要设置体长、得分相连的动态得分机制  
A：因为本游戏中体长不是不断增加的，贪吃蛇会因为饥饿不断缩减体长。长体长意味着更容易游戏失败，玩家可能求稳将体长控制得不是太长。体长越大得分越多能鼓励玩家维持长体长，是对保持高难度玩家的奖励  
Q：为什么设置速度与体长、得分相关的动态难度机制  
A：将速度与体长挂钩（身体越长速度越快）。这样在很长的体长意味着更高的得分背景下，因为速度和身体长的原因，意味着容易致死，但高速度能让你更快去吃东西，高风险高回报；此外得分达到一定会变阶段，目前不同阶段只有速度不同，可以简单理解为分越高速度也会越快，作为最基本的难度递增系统，希望玩家玩到一定阶段后能将基本难度提升几个档次，促进玩家游戏失败（避免永动机）
